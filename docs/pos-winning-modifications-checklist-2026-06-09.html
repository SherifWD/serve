Executive Checklist

Fix fake-table dependency for cashier-only takeaway, drinks, and counter-service shops.
Add backend support for no-table takeaway/delivery orders using branch_id and order_type.
Add send_to_cashier so cashier-created orders can enter payment flow immediately.
Add Flutter cashier counter-sale workflow with menu search, customer fields, cart quantity controls, submit, and queue refresh.
Add secured merchant CSV export endpoint and Flutter owner export button/dialog.
Run real API cycle: sale, payment, receipt PDF, and owner export.
Certify physical printers, cash drawers, tablets, and payment terminals in a lab.
Add true offline mode with local order/payment/print queue and conflict handling.
Add migration importers for Toast, Square, Clover, Foodics, spreadsheets, and common CSV exports.
Publish pricing, support SLA, hardware policy, payment policy, and data ownership pages.

Finished Steps

Area What changed Why it helps win Status
Backend order model flow No-table staff orders now support branch_id, order_type=takeaway, and nullable table_id. Only-takeaway restaurants, drinks shops, and counter cafes do not need fake tables. Done
Cashier shortcut send_to_cashier creates a no-table order directly in cashier state when payment should happen immediately. Matches the real cycle: cashier enters item, prints/charges receipt, receives payment, hands product to customer. Done
Flutter cashier Added Counter sale action, menu loader, search, customer name/phone, cart controls, quantity edits, and submit. Cashier-only shops can operate without waiter, table, or kitchen screens. Done
Data ownership Added secured CSV export for products, customers, orders, payments, receipts, and inventory items. Directly attacks lock-in complaints. Owners can prove data is portable. Done
Flutter owner Added Export CSV control with dataset dialog and browser download on web. Owner can self-serve exports instead of asking support or being trapped. Done
Production scenarios Scenario tests cover small cafe, big cafe split tender, drinks-only cashier takeaway, takeaway cafe with KDS, small restaurant, big restaurant table move, and customer takeaway pickup. Turns the sales claim into tested operational proof. Done
Receipts and print lifecycle Tests prove receipt PDF generation plus print job queue, claim, and printed status. Supports the cashier cycle expected by cafes and restaurants. Done
Cross-branch safety No-table access and receipt/payment access use branch/restaurant scope instead of table-only assumptions. Protects multi-branch operators and owner reporting. Done
Demo branch data Local demo stock was replenished for Janova branches 6 and 7 so live Flutter menu testing can show products. A demo branch with empty stock hides the menu and weakens sales demos even when the code is correct. Done

Winning Claims And Conditions

Safe claims now

  • Restaurant-first POS for cafes, takeaway shops, drinks shops, and dining rooms.
  • Supports waiter, kitchen/KDS, cashier, owner, and customer pickup flows.
  • Cashier-only counter sales work without fake tables.
  • Paid receipts and server print-job lifecycle are tested.
  • Owner CSV exports prove basic data portability.
  • Multi-branch scoping protects cashier and owner data boundaries.

Claims to avoid until gaps close

  • "Full Toast/Square/Clover replacement for every merchant."
  • "Offline like Toast or Square."
  • "Certified on all major printers, drawers, and terminals."
  • "One-day migration from any POS."
  • "Enterprise support SLA."
  • "Payment processing savings guaranteed."

Still Not Finished

P0 - Needed before broad competitive marketing

Gap Why competitors win today Winning condition Status
Payment processor adapters and terminal capture Toast, Square, Clover, Foodics, and Lightspeed sell integrated payments as a core promise. Certified adapters, terminal references, settlement reconciliation, refunds, tips, and failure handling. Open
Physical printer, kitchen printer, and cash drawer lab Competitors ship known hardware bundles and support scripts. Documented supported devices, setup wizard, rush-hour print test, drawer open test, retry and failed-printer escalation. Open
Offline/local-first mode Toast and Square sell outage survival as a major trust feature. Local order queue, local payment queue where allowed, local print queue, sync replay, conflict handling, offline receipt numbering. Open
Migration importers Merchants fear switching costs more than POS feature gaps. Import templates and validation for products, categories, modifiers, customers, inventory, and opening balances from Toast/Square/Clover/Foodics/spreadsheets. Open
Commercial trust pages Buyers compare total cost, support, cancellation, and hardware ownership before switching. Public pricing, support SLA, payment policy, hardware ownership, cancellation, data export, and onboarding process. Open
Flutter integration tests Widget/golden tests do not prove login, tap, submit, and download behavior end to end. Integration tests for cashier counter sale, payment, receipt action, owner export, waiter order, KDS ready, and customer pickup. Partial
