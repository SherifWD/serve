Offline And Day-Close Rules

Offline Rules

  • Queue order creation, KDS handoff, cashier handoff, payment attempts, pay order, print receipt, and print-job claim actions with client mutation IDs.
  • Use server updated_at as conflict winner unless the cashier explicitly resolves a difference.
  • Do not queue card captures unless the active provider explicitly permits offline authorization.
  • Use local receipt preview numbers until server receipt number is confirmed.

Shift Close Rules

  • Open shift with user, branch, device, opening cash, and note.
  • Close shift with expected cash, counted cash, variance, tips, service charge, and manager approval note.
  • Export payments, receipts, and cash drawer events with the end-of-day pack.
  • Escalate any failed print jobs, uncaptured attempts, or offline mutations before final close.
Status: The software foundation is done and live-tested. Full certification is complete only after physical devices and provider credentials are tested in a branch/lab matrix.
