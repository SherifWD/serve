Switching Import Pack

Practical migration templates and importer commands for restaurants moving from Toast, Square, Clover, Foodics, Lightspeed, spreadsheets, or ad hoc CSV exports.

Importer Command

Use the Laravel console importer to validate first, then write records after the owner approves the preview.

php artisan pos:import-switching-csv products docs/import-templates/products.csv --restaurant_id=1 --branch_id=6 --dry-run
php artisan pos:import-switching-csv products docs/import-templates/products.csv --restaurant_id=1 --branch_id=6

The command supports categories, products, customers, inventory-items, and opening-balances.

Templates

Migration Workflow

1. Export From Old POS

  • Ask for products/items, categories, customers, inventory, modifiers, and branch/location exports.
  • Keep original files unchanged in the onboarding folder.
  • Map each source column to a Janova template column.

2. Dry Run

  • Run --dry-run for every dataset.
  • Resolve missing columns, bad branches, duplicate SKUs, and malformed prices before import.
  • Send the dry-run summary to the owner for approval.

3. Import

  • Import categories before products.
  • Import products before inventory items if the stock is product-linked.
  • Import customers before live orders if loyalty opening points matter.

4. Reconcile

  • Owner reviews menu counts, prices, sample customer rows, and low-stock alerts.
  • Cashier and waiter run live browser checks before go-live.
  • Archive source files, imported templates, dry-run summaries, and final import logs.
Status: Migration importer support is now implemented in the repo through pos:import-switching-csv and the CSV templates above. Vendor-specific exports still need column mapping during onboarding because Toast, Square, Clover, Foodics, and spreadsheet exports vary by merchant configuration.
